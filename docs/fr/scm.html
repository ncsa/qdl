Vue d'ensemble

Ce projet utilise Subversion pour gérer son code source. Des instructions sur l'utilisation de Subversion peuvent être trouvées sur http://svnbook.red-bean.com/.

Accès Web

Ce qui suit est le lien online du dépôt de sources.

scm:svn:https://jeff92@svn.code.sf.net/p/cilogon/code/trunk/edu.uiuc.ncsa/ncsa-security-all/website

Accès de façon anonyme

Le dépôt d'archive Subversion de ce projet peut être consulté de façon anonyme avec les instructions suivantes :

$ svn checkout https://svn.code.sf.net/p/cilogon/code/trunk/edu.uiuc.ncsa/ncsa-security-all/website website

Accès pour les dévelopeurs

Chacun peut accéder au dépôt d'archive de Subversion via HTTP, mais seuls les committers doivent accéder au dépôt d'archive via HTTPS.

$ svn checkout https://svn.code.sf.net/p/cilogon/code/trunk/edu.uiuc.ncsa/ncsa-security-all/website website

Pour commiter les changements dans le dépôt, exécuter la ligne de commande suivante (Subversion pourra vous demander votre mot de passe)

$ svn commit --username your-username -m "A message"

Accès derrière un firewall

Pour les utilisateurs qui sont coincés derrière un firewall qui bloque l'accès de HTTP au dépôt de Subversion, vous pouvez essayer d'y accéder avec la connection développeur :

$ svn checkout https://svn.code.sf.net/p/cilogon/code/trunk/edu.uiuc.ncsa/ncsa-security-all/website website

Accès avec un proxy

Le client de subversion peut accéder par proxy, si vous le configurez comme suit. D'abord, éditez votre dossier de configuration de "servers" pour indiquer quel proxy utiliser. Ce fichier est situé différemment selon votre système d'exploitation. Sur Linux ou Unix, il est situé dans le répertoire "~/.subversion". Sur Windows, il se situe dans "%APPDATA%\Subversion". (essayez de taper "echo %APPDATA%", notons que ce répertoire est caché.)

Il y a des commentaires dans le fichier qui expliquent comment faire. Si vous n'avez pas ce fichier, téléchargez la version la plus récente de Subversion et tapez n'importe quelle commande ; ceci créera les fichiers de configuration.

Exemple : Editer le fichier 'servers' et ajouter quelque chose du genre :

[global]
http-proxy-host = your.proxy.name
http-proxy-port = 3128
